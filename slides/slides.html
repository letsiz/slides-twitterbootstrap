Was stellt es dar


  • 2010 von Mark Otto und Jacob Thornton bei Twitter als eine Art Sandbox / Online Styleguide für interne Anwendungen entwickelt
    … für eine homogene/konsistente Darstellung mit geringem Wartungsaufwand
  • 2011 Open Source, 2012 erfolgreichstes Projekt auf Github
  • adaptive RWD (Mobile First: Dropdown-Menü, pixelbasiert)
  • Browserkompatibilität ab IE 8, Progressive Enhancement
  • große Verbreitung / Community = viele aktiv gepflegte und umfangreiche Plugins wie z. B. Datepicker, Chosen, jQuery Upload, Uniform bzw. Fuel UX und viele viele weitere hier auf Github

Pro

Browserkompatibilität, Plugins, Integration

Pro


  • zeitsparende/beschleunigte Frontend-Entwicklung
    • modularer Aufbau von HTML/CSS
    • grundlegende Stildefinitionen für alle wichtigen HTML-Elemente enthalten, CSS-Reset/Normalize, Grid-System, div. Helper-Klassen und viele weitere Elemente
    • optionale JavaScript-Plugins auf Basis von jQuery für Tooltips, Modals und Dropdowns
    • Browserkompatibilität … nur noch geringfügig Browsertests der unterstützen Browser und Geräte nötig

Pro


  • möglicher Einsatz von Drag & Drop Interface Buildern
  • gute Konfigurierbarkeit … u. a. Module durch »Custom download« an-/abwählbar
  • Integration in bekannte Web Application Frameworks wie Rails (über RubyGems) vorhanden

Contra

Twitter-Styled-Design, Dokumentation

Contra


  • besondere Anforderungen im Gestaltungsprozess
    • Twitter-Styled …
      festes 12er-Raster usw. bedeuten geringere gestalterische Spielräume und ein weniger einzigartiges Design?!
    • RWD für neue Layouts bzw. Elemente

Contra


  • Einarbeitung in umfangreiche Dokumentation nötig, neues Konzept verstehen & verinnerlichen
  • umfangreiche Assets mindern Performance und Flexibilität
  • lokales Entwicklungssystem LESS/SASS benötigt
  • riesiger Pool an zur Auswahl stehender Elemente, Floats und Grids verleitet zu Spielereien
  • HTML5 und CSS3 vorausgesetzt, Polyfills/Shims für IE8 u. U. benötigt
